# E-Commerce Platform (Microservices)

Пет-проект на Spring Boot, в котором я собрал **scalable e-commerce platform** в микросервисной архитектуре.  
Основная цель проекта — не просто сделать интернет-магазин, а **понять микросервисы, Kafka, саги, идемпотентность, компенсации и Docker-архитектуру** на практике.

## Что внутри

- **6 микросервисов**
- **Apache Kafka** для взаимодействия между сервисами
- **PostgreSQL** для каждого микросервиса отдельно
- **Liquibase** для миграций
- **Docker Compose** для запуска всей системы одной командой
- **Testcontainers** для интеграционных тестов
- **Stripe sandbox** для платежей
- **Mail sandbox / SMTP** для отправки писем
- **Собственная общая библиотека `core`**, опубликованная как зависимость в GitHub Packages

## Архитектура

Проект построен вокруг событийного обмена сообщениями через Kafka.

Внутри сервисов я использовал:

- **idempotency** — защита от повторной обработки сообщений;
- **transactional processing** — работа в транзакциях;
- **retryable / not retryable exceptions** — разделение ошибок на временные и фатальные;
- **dead letter topic (DLT)** — перенос проблемных сообщений в отдельный топик;
- **saga pattern** — оркестрация основного и компенсационного сценариев;
- **messageId header** — для отслеживания и защиты от дублей.

## Микросервисы

### `OrderMS`
Центральный сервис оркестрации заказа.

Что делает:
- создает заказ;
- ведет историю состояний заказа;
- запускает forward / rollback saga;
- отправляет команды в другие сервисы;
- при успехе уведомляет пользователя по email.

### `ProductMS`
Сервис каталога и склада.

Что делает:
- резервирует товар под заказ;
- отменяет резерв;
- хранит историю резервирования;
- публикует события об успехе / ошибке.

### `PaymentMS`
Сервис оплаты.

Что делает:
- создает платеж через Stripe;
- делает возврат;
- работает с пользовательской оплатой и пополнением баланса;
- публикует события в Kafka для дальнейшего движения саги.

### `UserMS`
Сервис пользователей и баланса.

Что делает:
- создает пользователя;
- пополняет баланс;
- списывает баланс;
- отменяет списание при компенсации;
- публикует события для платежного и email-сценариев.

### `ShoppingCartMS`
Сервис корзины.

Что делает:
- добавляет товары в корзину;
- изменяет количество;
- удаляет позиции;
- оформляет покупку из корзины и публикует событие для `OrderMS`.

### `EmailNotificationMS`
Сервис уведомлений.

Что делает:
- отправляет письма через SMTP;
- слушает Kafka-сообщения;
- обрабатывает обычные и компенсационные email-сценарии;
- сохраняет обработанные сообщения для идемпотентности;
- отправляет проблемные сообщения в DLT.

### `core`
Общая библиотека для всех микросервисов.

Содержит:
- общие DTO для команд и событий;
- базовые исключения;
- базовую JPA-сущность для idempotency;
- общий контракт между сервисами.

## Как это работает

Сценарий покупки проходит как цепочка событий:

1. `ShoppingCartMS` создает событие на оформление заказа.
2. `OrderMS` начинает оркестрацию.
3. `ProductMS` резервирует товар.
4. `PaymentMS` проводит оплату.
5. `UserMS` списывает баланс.
6. `OrderMS` подтверждает заказ.
7. `EmailNotificationMS` отправляет письмо пользователю.

Если на одном из шагов возникает ошибка, запускается **компенсационная сагa**, которая откатывает уже выполненные действия:
- отмена резерва товара;
- возврат оплаты;
- отмена списания баланса;
- смена статуса заказа;
- уведомление пользователя об ошибке.

## Особенности реализации

- у каждого микросервиса своя база данных;
- у каждого микросервиса свой Dockerfile;
- каждый сервис собирается через multi-stage build;
- Kafka-кластер поднимается из 3 брокеров;
- все сервисы и БД изолированы сетями Docker;
- БД доступны только микросервисам;
- Kafka-брокеры доступны друг другу и микросервисам;
- интеграционные тесты написаны через Testcontainers;
- общая библиотека `core` подключается как внешняя зависимость из GitHub Packages.

## Как запустить проект

### 1. Подготовь `.env`

В корне проекта нужен файл `.env` с переменными окружения.

Пример:

```env
KAFKA_CLUSTER=your-kafka-cluster-id
POSTGRES_USER=postgres
POSTGRES_PASSWORD=root
GITHUB_TOKEN=your_github_packages_token
SPRING_MAIL_USERNAME=your_email@gmail.com
SPRING_MAIL_PASSWORD=your_smtp_app_password
STRIPE_API_KEY=your_stripe_test_key
```

`GITHUB_TOKEN` нужен для скачивания общей библиотеки `core` из GitHub Packages.

### 2. Собери и запусти все сервисы

```bash
docker compose up --build
```

### 3. Открой сервисы

- EmailNotificationMS — `http://localhost:8081`
- OrderMS — `http://localhost:8082`
- PaymentMS — `http://localhost:8083`
- ProductMS — `http://localhost:8084`
- ShoppingCartMS — `http://localhost:8085`
- UserMS — `http://localhost:8086`

Kafka brokers:
- `localhost:9094`
- `localhost:9095`
- `localhost:9096`

### 4. Останови систему

```bash
docker compose down
```

Чтобы удалить еще и тома с данными:

```bash
docker compose down -v
```
